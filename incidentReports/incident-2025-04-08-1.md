# Incident: 2026-04-08 09-36-00

## Summary

```md

Between the hour of 9:36 and 12:24 on April 8, 2026, all users encountered an inability to order a pizza. The event was triggered by a system update at 10:30pm the previous night. The update contained a testing software that would manually cause orders to fail as a test of our website's alert detection.

A bug in this code caused every order of a pizza to fail. The event was detected by Grafana. The team started working on the event by opening the link to resolve the issue. This severe incident affected 100% of users.

```

## Detection

```md

This incident originally did not set off the alert on account of it not having a proper threshold for failed orders. The alert was quickly created when upon monitoring Grafana, the failed orders and set orders seemed to match. 

Whent the alert was fixed, Ethan Moreno, who was on call, was paged immediately after. 

Multiple alerts will be made that not only compare a metric to a general number but also to each other. 
```

## Impact

```md

For 2hrs and 48 minutes between 3:36 UTC and 6:24 UTC on 04/08/2026, all of our users experienced this incident.

No support tickets were submitted.
```

## Timeline

```md
All times are UTC.

- _5:30_ - Testing Update Set
- _3:36_ - Orders begin to fail 
- _6:02_ - Incident is noted

```

## Response

```md

After monitoring page at 6:02 UTC, Ethan Moreno came online and resolved the incident at 6:24 UTC.

```

## Root cause

```md

A bug in connection pool handling led to leaked connections under failure conditions, combined with lack of visibility into connection state.
```

## Resolution

```md

By adjusting alerts so when various individual metrics are at different relative positions, the on-call engineer(s) is notified.  
```
